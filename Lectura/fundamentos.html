Fundamentos BBDD

Creación de una Base de Datos

Antes de trabajar con una base de datos, es necesario crearla y definir sus tablas junto con sus atributos.

Crear una Base de Datos

Para crear una base de datos en SQL, usamos el siguiente comando:

CREATE DATABASE Tienda;

Crear Tablas y Definir Atributos

Después de crear la base de datos, debemos definir las tablas y sus atributos.

CREATE TABLE Clientes (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Email VARCHAR(100)
);

CREATE TABLE Productos (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Precio DECIMAL(10,2)
);

CREATE TABLE Pedidos (
    ID INT PRIMARY KEY,
    ClienteID INT,
    Fecha DATE,
    FOREIGN KEY (ClienteID) REFERENCES Clientes(ID)
);

Claves Primarias y Foráneas

Las claves primarias y foráneas son fundamentales para la integridad de los datos en bases de datos relacionales.

Clave Primaria

Una clave primaria es un identificador único para cada fila en una tabla. No puede contener valores nulos ni repetidos.

CREATE TABLE Clientes (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Email VARCHAR(100)
);

Clave Foránea

Una clave foránea es un campo en una tabla que hace referencia a la clave primaria de otra tabla.

CREATE TABLE Pedidos (
    ID INT PRIMARY KEY,
    ClienteID INT,
    FOREIGN KEY (ClienteID) REFERENCES Clientes(ID)
);

Normalización

El proceso de normalización se usa para minimizar la redundancia y mejorar la integridad de los datos.

Ejemplo de Normalización

Supongamos una tabla sin normalizar:

Pedidos(ID, Cliente, Dirección, Producto, Precio, Cantidad)

Después de aplicar la normalización:

Clientes(ID, Nombre, Dirección)
Pedidos(ID, ClienteID, ProductoID, Cantidad)
Productos(ID, Nombre, Precio)

Lenguajes de Consulta

Los lenguajes de consulta en bases de datos se dividen en diferentes categorías según su propósito.

Lenguaje de Definición de Datos (DDL)

Usado para definir la estructura de la base de datos.

CREATE TABLE Productos (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Precio DECIMAL(10,2)
);

Lenguaje de Manipulación de Datos (DML)

Permite la inserción, actualización y eliminación de datos.

INSERT INTO Productos (ID, Nombre, Precio) VALUES (1, 'Laptop', 1200.50);
UPDATE Productos SET Precio = 1150.00 WHERE ID = 1;
DELETE FROM Productos WHERE ID = 1;

Lenguaje de Control de Datos (DCL)

Gestiona permisos y accesos a la base de datos.

GRANT SELECT ON Productos TO Usuario1;
REVOKE SELECT ON Productos FROM Usuario1;

Lenguaje de Control de Transacciones (TCL)

Maneja la ejecución de transacciones.

START TRANSACTION;
UPDATE Productos SET Precio = 900.00 WHERE ID = 1;
COMMIT;

Índices en Bases de Datos

Los índices mejoran el rendimiento de las consultas en bases de datos al acelerar la búsqueda de datos.

Ejemplo de Creación de Índice

CREATE INDEX idx_producto_nombre ON Productos(Nombre);
